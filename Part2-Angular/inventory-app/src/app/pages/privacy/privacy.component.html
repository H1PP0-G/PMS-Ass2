Privacy & Security Analysis 🛡️

1. Data Persistence & Privacy

This project adopts "session-based storage". All inventory data is only saved in the browser's memory and will not be uploaded to any backend server. This means that once the browser tab is closed, the data will be automatically cleared, thereby maximizing the protection of user privacy and avoiding the risk of data leakage.

2. Input Validation (Security)

To prevent the input of illegal data, we have employed Angular's two-way binding and TypeScript's type checking. All numeric fields, such as price and quantity, have undergone strict type validation to ensure that no malicious scripts or invalid data are executed.

3. XSS (Cross-Site Scripting) Prevention

By using Angular's interpolation syntax ({{ '{{' }} value {{ '}}' }}), the system automatically escapes the rendered content, preventing potential cross-site scripting attacks and ensuring the security of the user interface.
